Executive Summary¶

CodeGraph is the only solution combining:

  • CPG-based analysis with taint-verified vulnerability detection

  • SIEM integration in three formats (Syslog, CEF, LEEF)
  • Russian LLMs (GigaChat, Yandex AI Studio) for 152-FZ compliance
  • Multi-criteria hypothesis validation to reduce false positives

CodeGraph Unique Advantages¶

1. Only Platform with Integrated DLP¶

                    CodeGraph                    Competitors
                    ─────────                    ───────────
User Query ──► [DLP Scanner] ──► LLM       User Query ──► LLM
                    │                                │
                    ▼                                ▼
              Block/Mask/Log                   No protection

Why This Matters: - Prevents secret leakage through LLM prompts - Masks PII in LLM responses - GDPR and data protection compliance

2. Only Platform with SIEM in Three Formats¶

CodeGraph GitHub Copilot Sourcegraph CodeScene

3. Only Platform with Vault Integration¶

# CodeGraph
vault:
  enabled: true
  auth_method: approle
  auto_rotation: true

# Competitors: env vars or plaintext configs only

Why This Matters: - Centralized secrets management - Automatic API key rotation - Kubernetes-native authorization

4. Taint-Verified Vulnerabilities¶

CodeGraph:
Source ──[data flow]──► Sink = CONFIRMED vulnerability

Traditional SAST:
Pattern match = POSSIBLE vulnerability (high false positive rate)

Results: - 100% CVE detection rate (3/3 target CVEs) - 60%+ false positive reduction - Prioritization by actual risk

5. Multi-Criteria Hypothesis Validation¶

Priority Score = (CWE Frequency × 0.40)
               + (Attack Similarity × 0.30)
               + (Codebase Exposure × 0.30)

Competitors: Static severity without codebase context

6. Agent Client Protocol (ACP) Support¶

                    CodeGraph                    Competitors
                    ─────────                    ───────────
IDE ──► [ACP Agent] ──► CodeGraph API       IDE ──► Proprietary API
              │                                      │
              ▼                                      ▼
     stdio / HTTP / WebSocket              Limited integration

Supported IDEs: - Zed (native ACP support) - JetBrains IDEs (IntelliJ, PyCharm, WebStorm, etc.) - VS Code (via extension) - Any ACP-compatible editor

Transport Options: | Transport | Use Case | |-----------|----------| | stdio | Local subprocess (IDE spawned) | | HTTP | Remote REST API | | WebSocket | Real-time streaming |

Why This Matters: - Native integration with modern IDEs (Zed, Cursor, etc.) - Open protocol — not vendor lock-in - Real-time streaming with session updates - MCP (Model Context Protocol) support for tool integration
